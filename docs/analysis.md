Ανάλυση του Προβλήματος

Καλούμαστε να υλοποιήσουμε ένα πρόγραμμα το οποίο λαμβάνει ως είσοδο ένα αρχείο κειμένου και παράγει ένα νέο αρχείο με το ίδιο περιεχόμενο αλλά διαφοροποίημενο σύμφωνα με συγκεκριμένους κανόνες(κεφαλαία, πεζά, μετατροπές αριθμών, σωστή χρήση σημείων στίξης κλπ). Στόχος είναι η παραγωγή ενός γραμματικά ορθού κειμένου σύμφωνα με αυτούσ τους κανόνες.

Καταγραφή κανόνων:

1)( hex)          ->  Μετατρέπει  τον προηγούμενο αριθμό από δεκαεξαδικό σε δεκαδικό
2) (bin)          ->  Μετατρέπει  τον προηγούμενο αριθμό από δυαδικό σε δεκαδικό
3) (up)           ->  Κάνει την προηγούμενη λέξη κεφαλαία
4) (low)          ->  Κάνει την προηγούμενη λέξη πεζά
5) (cap)          ->  Κάνει την προηγούμενη λέξη να ξεκινά με κεφαλαίο
6) (up,n)         ->  Κάνει τις προηγούμενες n λέξεις κεφαλαίες
7) (low,n)        ->  Κάνει τις προηγούμενες n λέξεις πεζές
8) (cap,n)        ->  Κάνει τις προηγούμενες n λέξεις κεφαλαιοποιημένες
9)Σημεία στίξης ->  Τα σημεία στίξης κολλάνε στην προηγούμενη λέξη και αφήνουν ένα κενό μετά
10)Απόστροφοι  -> Κολλάνε στις λέξεις χωρίς κενά
11)Άρθρο a/an   -> Το a γίνεται an αν ακολουθεί λέξη που αρχίζει από φωνήεν ή “h”


Σύγκριση αρχιτεκτονικών Pipeline/FSM

Pipeline:

Στην pipeline αρχιτεκτονική το κείμενο περνάει από μία ακολουθία σταδίων. Κάθε στάδιο εκτελεί μία ανεξάρτητη μετατρόπη.Αυτό κάνει την υλοποίηση πιο εύκολη και απλή όπως κάνει και την προσθήκη ενός νέου μετασχηματισμού εύκολη. Τα μειονεκτήματα της είναι ότι είναι δύσκολο να διαχειριστεί δύο κοντινούς μετασχηματισμούς και το κυριότερο μειονέκτημα είναι ότι χρειάζεται πολλαπλά περάσματα στο κείμενο για να κάνει τις μετατροπές άρα δεν είναι πολύ αποδοτικό.

FSM:

Στην FSM προσέγγιση διαβάζουμε το κείμενο λέξη προς λέξη και ανάλογα μ ετην λέξη που βρίσκουμε μεταβαίνουμε σε άλλη κατάσταση . Κάθε κατάσταση αποφασίζει πως θα χειριστεί την τρέχουσα λέξη ή τι μετατροπές θα κάνει, όπως και σε ποια κατάσταση θα μεταβεί. Τα πλεονεκτήματα αυτης της προσέγγισης είναι η πιο ελεγχόμενη ροή καθώς κάθε εντολή επηρεάζει τις ακριβώς προηγούμενες λέξεις, ο ευκολότερος χειρισμός για σύνθετους συνδυασμούς και ότι γίνεται μόνο μία διέλευση του κειμένου, άρα σε μεγάλα κείμενα είναι πιο αποδοτική.  Τα μειονεκτήματα αυτής της προσέγγισης είναι ότι είναι πιο δύσκολη η υλοποίηση της και ότι απαιτεί ξεκάθαρο ορισμό των καταστάσεων καθως και των μεταβάσεων.

Για τους παραπάνω λόγους θα επιλέξω την FSM προσέγγιση καθώς είναι πιο αποδοτική και προσφέρει καλύτερο έλεγχο στην ροή του προγράμματος όπως και καλύτερη αντιμετώπιση σύνθετων περιπτώσεων.

FSM ΜΟΝΤΕΛΟ

Για το συγκεκριμένο πρόγραμμα οι καταστάσεις που θα χρησιμοποιηθούν θα ειναι οι:
READING
COMMAND
QUOTE 
PUNCTUATION


Ακολουθεί ένας πίνακας μεταβάσεων για τις καταστάσεις αυτές:

STATE       	       TRIGGER	                  NEXT STATE
READING	         hex,bin,up,low,cap,”a”	            COMMAND
READING	               ,.!?;:	                     QUOTE
READING	                 ‘                        PUNCTUATION
COMMAND	         End of transformation	            READING
QUOTE	         End of transformation	            READING
PUNCTUATION	     End of transformation	            READING
